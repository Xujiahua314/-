## 📋 项目简介

本系统是一个基于 **C 语言**和**图数据结构**的景区路径规划工具，采用**邻接表**存储景点和路径信息，支持导游线路生成、最短路径查询等功能。

## 🗂️ 项目结构

```
景区路径规划系统/
├── graph.h          # 头文件：数据结构定义 + 函数声明
├── graph.c          # 功能实现：图的所有核心算法
├── main.c           # 主函数：控制台交互菜单
├── README.md        # 项目说明文档

```

## 🎯 功能列表

| 序号 | 功能 | 算法/实现 | 说明 |
|------|------|-----------|------|
| 1 | 创建景区景点分布图 | 邻接表 + 邻接矩阵 | 添加景点和路径，构建图结构 |
| 2 | 展示景点分布图 | 邻接矩阵输出 | 以矩阵形式展示所有景点距离，∞=32767 |
| 3 | 输出导游线路图 | DFS 深度优先遍历 | 从指定景点出发，覆盖所有可达景点 |
| 4 | 判断回路 | 重复访问检测 | 检测导游线路中是否存在回路并输出 |
| 5 | 查询最短路径 | Dijkstra 算法 | 任意两景点间的最短路径及距离 |
| 6 | 最小生成树 | ✨ Prim 算法（扩展） | 道路建设优化方案 |

## 🛠️ 技术要点

- **数据结构**：邻接表（主存储）+ 邻接矩阵（展示用），支持最多 50 个景点
- **无边距表示**：使用 32767（∞）表示两景点之间没有直接连通路径
- **DFS 导游线路**：深度优先遍历，包含回溯路径，模拟导游带领游客全程游览
- **Dijkstra 算法**：贪心策略，时间复杂度 O(n²)，输出完整路径和分段距离
- **Prim 算法**：用于生成最小生成树，可用于景区道路建设成本优化

## 📝 使用说明

1. 启动程序后自动加载预设数据（8景点9条边）
2. 在菜单中选择对应数字（0-9）执行功能
3. 可手动添加景点、添加路径（边）
4. 生成导游线路后方可检测回路
5. 选择 9 退出系统
